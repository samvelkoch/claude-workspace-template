## Git-протокол

- Никогда не коммитить в main напрямую
- Ветка: `[experiment|feature|fix]/AAI-XXXX-краткое-описание`
- Параллельная работа: `claude --worktree AAI-XXXX-описание` для каждого эксперимента
- Один коммит = одна логическая единица (формат: `[experiment|feature|fix|refactor] описание`)
- Перед коммитом: baseline check прошёл + результат записан в PROGRESS.md
- Не коммитить: raw данные, credentials, ноутбуки с полным выводом, >10MB

**Handoff-коммит** (при передаче хода другой машине):
1. Обнови `## Handoff` в PROGRESS.md: новый `Ход: <to-host-id>`, очисти "В процессе".
2. `git commit -m "handoff: <FROM>→<TO> — [что сделано, что дальше]"`
3. `git push`

Тег "В процессе" очищается только при handoff-коммите, не раньше.

## Handoff и worktrees

Решают разные задачи, не путать:

- **Handoff** — передача активной работы **между машинами** через git push/pull. Поле `## Handoff → Ход:` показывает чья сейчас очередь действовать.
- **Worktrees** — параллельные эксперименты **на одной машине** (разные ветки одного репозитория). Не привязаны к Handoff.

Машина с активным `Ход:` может вести несколько worktrees параллельно. Передача хода другой машине = пушнуть всё нужное (включая ветки worktrees если они должны переехать) + handoff-коммит.

## Параллельная работа

Что легко параллелизируется:
- Разные группы фичей / модели / сегменты данных

Что блокирует всё (не параллелить пока не решено):
- Утечка данных, изменение схемы данных, изменение таргета

Claiming при параллельной работе — записать в PROGRESS.md "В работе":
```
[2026-03-25 14:30] @worktree-features: lag_7, lag_14, rolling_mean
[2026-03-25 14:30] @worktree-catboost: CatBoost baseline, тот же feature set
```

Передача контекста субагентам — явно в промпт: статус из PROGRESS.md,
dead ends, oracle-проверки, куда записать результат.

## Синхронизация документов

Что обновлять при значимых изменениях (значимое = меняет
поведение / интерфейс / интерпретацию данных, не косметика):

| Изменение | PROGRESS.md | CLAUDE.md | data_schema.md | decisions.md | README.md |
|-----------|-------------|-----------|----------------|--------------|-----------|
| Изменение схемы данных / интерфейса | ✓ Лог результатов | ✓ Ключевые файлы (если новый модуль) | ✓ | — | — |
| Смена точки отсчёта / критерия приёмки | ✓ Baseline | ✓ Baseline | — | ✓ | — |
| Архитектурное решение | ✓ Сделано | ✓ Архитектурные решения | — | ✓ | — |
| Найден новый запрет / антипаттерн | ✓ Dead ends | ✓ Запреты проекта | — | ✓ (опц.) | — |
| Новая команда / точка входа | — | ✓ Ключевые файлы | — | — | ✓ |
| Новая переменная окружения | — | — | — | — | ✓ + `.env.example` |

При сомнении — обновлять оба. Расхождение CLAUDE.md и кода = агент будет действовать по устаревшему контракту.
