# Роли агентов

Для сложных задач запускать специализированных агентов с разными worktrees.

Типичные роли (адаптировать под проект):

- **researcher** — исследует одно направление / гипотезу / источник
- **implementer** — реализует конкретный модуль / фичу по спеке
- **validator** — проверяет корректность (тесты, инварианты, схемы)
- **reviewer** — проверяет код против правил проекта и плана
- **documenter** — обновляет PROGRESS.md, пишет summary

Назначать разные файлы / модули разным агентам — не давать двум агентам
трогать один файл одновременно.

Координатор обновляет PROGRESS.md после завершения каждого субагента.
