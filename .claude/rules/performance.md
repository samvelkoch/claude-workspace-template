# Эффективность на масштабе

При работе с большими объёмами данных или долгими прогонами избегать
неэффективных конструкций.

- Использовать batch / vector / native операции вместо построчных циклов
- Применять встроенные операции языка/библиотек вместо ручных реализаций
- Если нужен цикл по большому объёму — сначала проверить есть ли batch-альтернатива

Конкретные паттерны зависят от стека:
- Табличные данные: pandas/polars/numpy векторизация, groupby, boolean indexing
- Базы данных: один запрос с агрегацией вместо N запросов в цикле
- API: batch endpoints вместо N последовательных вызовов

Не оптимизировать преждевременно — сначала измерить узкое место.
Если объём небольшой и читаемость важнее — допустим явный цикл с пометкой почему.
