# Стандарт документации

Писать для следующей сессии (агент без памяти), не для пользователей.

Каждая нетривиальная функция документирует:
- Какую бизнес-концепцию отражает
- Формы и типы входных/выходных данных
- Почему именно это кодирование
- Известные особенности данных влияющие на фичу
- Риски (например, утечка через будущие данные)

Комментарии объясняют ПОЧЕМУ, не ЧТО.
