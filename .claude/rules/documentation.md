# Стандарт документации

Писать для следующей сессии (агент без памяти), не для пользователей.

Каждая нетривиальная функция документирует:
- Какую концепцию / бизнес-операцию отражает
- Контракт: типы входов / выходов, инварианты
- Почему выбрана именно эта реализация
- Известные особенности / угловые случаи входных данных
- Риски (если есть): что может пойти не так и при каких условиях

Комментарии объясняют ПОЧЕМУ, не ЧТО.
